## 1. 宇宙核心概念：LL 是什麼？

### 1.1 LeafLune 的本質

- LeafLune（LL）是一個：
  - 「寓教於樂」的 **Edutainment 宇宙**
  - 用 **強化式學習（RL）世界觀** 來看待人生、學習、遊戲、創業的系統
  - 結合：
    - 教育（學習與成長）
    - 遊戲（挑戰與樂趣）
    - AI（工具與同伴）
  的長期生態系

### 1.2 三層向度：外擴 × 內聚 × 哲學

- 向外（實體化）：  
  把宇宙具現成品牌、平台、課程、遊戲、工具。

- 向內（系統化）：  
  把腦中的各種藍圖全部結構化、SOP 化、模組化，  
  變成未來 AI 工廠可以重複使用的流程。

- 哲學層（RL 世界觀）：  
  - 用「狀態 / 動作 / 回饋」（State / Action / Reward）解構人生
  - 把「人生觀 / 價值觀 / 世界觀」用 RL 語言重新定義
  - 讓整個 LL 宇宙有一套統一的核心邏輯

## 2. 品牌星群：五大品牌＋兩大核心系統

### 2.1 品牌層（對外）

1. **宣學習 XuanXuexi（XX）**
   - 定位：兒童 / 青少年語言學習品牌
   - 主題：語言卡牌、繪本、英語營隊、AI 助教
   - 代表產品：
     - Pinyin Poker 64
     - Sentence Summoner
     - AIGC 協作語言課程

2. **LeafLune（LL）**
   - 定位：主品牌 / 宇宙名稱
   - 主題：Edutainment、RL 哲學、人生系統、AI 工廠
   - 代表內容：
     - LL 宇宙世界觀
     - RL 人生觀
     - 大型藍圖與策略文件（本文件即其一）

3. **ReinRoom 強化教室（RR）**
   - 定位：RL 教學與實驗平台
   - 主題：Blockly + RL + Matter.js + 遊戲環境
   - 代表內容：
     - ReinforceLab 平台
     - PolicyPackage（副檔名 `.pp`）

4. **GameGeek 遊戲極客（GG）**
   - 定位：多平台遊戲控制器 / 互動介面
   - 主題：Web 遊戲手把、通訊協定、玩家輸入層
   - 代表內容：
     - GG ↔ 遊戲 通訊標準
     - Emoji 遊戲控制面板

5. **玄機界域 StrategySpace（SS）**
   - 定位：戰略對戰與競技空間
   - 主題：
     - 對戰遊戲
     - RL Agents vs Player
     - 賞金任務、比賽
   - 代表內容：
     - ShadowPath / Shadow Protocol 類戰略遊戲
     - RL 競技賽事規格

### 2.2 系統層（對內）

6. **矩陣總管 MatrixManager（MM）**
   - 定位：任務、資料與 UI 的中樞管理 AI
   - 功能：
     - 統整任務表、狀態、優先級
     - 把底層 DB 與人類 UI 中間加上一層 AI 代理
   - 三大 UI 原則：
     1. 設計完善的系統，人類不需要直接碰底層資料庫
     2. 畫面以方便人類看的方式呈現人類需要看的資料為優先
     3. UI 是「跟 AI 說話」的管道，而不是 DB 控制台

7. **司簿星（TaskTable / 任務 AI）**
   - 定位：LL 宇宙的賬簿與任務管理 AI
   - 功能：
     - 記錄任務、進度
     - 控管個人 / 專案 / 品牌的工作流
     - 未來會與 MM、HackMD 深度整合

> 待補：  
> - 為每一個品牌與系統寫獨立 spec 文件  
> - 以 `LL_Brand_XX_spec_v1.md` 等方式在 HackMD 中存放

    - 說明這份 SOP 要解決什麼問題
    - 適用範圍（專案 / 品牌 / 對象）

    ## 2. 輸入（Input）
    - 必要資料：
    - 檔案格式：
    - 來源（HackMD / PDF / 實體筆記…）：

    ## 3. 工具與 AI 角色（Tools & AI Roles）

    ## 4. 步驟流程（Steps）

    ## 5. 輸出（Output）
    - 產出的檔案 / 文件 / 網頁
    - 存放位置（HackMD 路徑）

    ## 6. 注意事項（Notes）
    - 隱私與授權
    - 常見陷阱 / Bug 記錄

    ## 7. 版本紀錄（Changelog）
    - v1 / [日期] / [簡短備註]

### 4.3 IE-3｜RL 哲學與人生觀系統化（概略）

詳內容應另建 `LL_RL_Philosophy_v1.md`，此處只記大綱：

- 「見山是山 / 見山不是山 / 見山又是山」對應 RL 思維層級：
  - 第一層：世界是一堆事件（尚未建模）
  - 第二層：世界是一個 MDP / SAR 流（開始看見底層結構）
  - 第三層：RL 模型內化成直覺，看世界仍是「山」，但眼裡已經內建 State / Action / Reward

- 核心觀念：
  - 人生是一個有限時間的 RL 過程
  - Reward 不等於短期爽感，而是長期 Value Function
  - Policy 的學習來源：
    - 自己的經驗
    - 他人的經驗（模仿學習）
    - AI 與工具提供的策略建議

## 5. 版本與里程碑控制

### 5.1 文件版本策略

- 核心文件（本檔、品牌星群、RL 哲學…）：
  - 大改 → v2, v3…
  - 每次大改簡述變動內容

- 專案規格書：
  - 依專案節奏更新（例如 RR v1 → v1.1 → v2）

### 5.2 宇宙建設階段（粗略 Roadmap）

v1 階段目標：

- 明確定義所有品牌 / 系統角色
- HackMD 知識庫基本結構完成
- 3～5 個核心 SOP 定稿
- 至少兩個專案有可用 MVP（例如：ReinRoom、GameGeek v0.1）

v2 之後：

- 多 AI 會議 / Section9 Protocol 實作
- API 型 AI 工廠成形
- CubicCraft 前期任務開始
- 玄機界域 SS 初版競技場上線
